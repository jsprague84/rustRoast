## Codebase Patterns
- Migrations use `include_str!("../migrations/xxx.sql")` with split-by-semicolon execution and silent error handling (tables may already exist)
- Use `IF NOT EXISTS` for all CREATE TABLE/INDEX/TRIGGER statements for idempotency
- Triggers use `CREATE TRIGGER IF NOT EXISTS` syntax for updated_at auto-update
- Foreign keys use `ON DELETE CASCADE` for child tables, `ON DELETE SET NULL` for optional references
- AppState is in main.rs with clone-able fields (Arc<RwLock<...>>, SqlitePool, etc.)
- RoastSessionService follows the service pattern: struct with SqlitePool, async methods returning Result
- Route handlers are defined in main.rs (monolithic file) — control, session, profile, telemetry handlers all inline
- Models (request/response structs) are in models.rs with Serialize/Deserialize derives
- Tests use in-memory SQLite with the same migration pattern
- serde_json::Value in FromRow structs requires: sqlx `json` feature in Cargo.toml + `#[sqlx(json)]` attribute on the field
- When binding serde_json::Value to queries, convert to string first: `serde_json::to_string(&value)?`
- set_register_map uses a transaction: DELETE all existing + INSERT new entries for atomic replacement
