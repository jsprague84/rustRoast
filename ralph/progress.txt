## Codebase Patterns
- Migration files in `crates/server/migrations/` — each ALTER TABLE must be a separate statement (SQLite fails silently if column exists)
- Migrations 001 and 003 were included in `init_db()` but 002 (roast_events) was missing — fixed; always include all migration files
- Tests use in-memory SQLite via `setup_test_db()` in services.rs — must mirror the same migration list as `init_db()`
- `RoastSession` model uses `#[derive(FromRow)]` — fields MUST match column order/names in `roast_sessions` table
- Frontend types: `RoastSession` interface is in `$lib/api/client.ts`, `SessionWithTelemetry` extends it in `$lib/types/session.ts`
- `query_as::<_, RoastSession>` with `RETURNING *` requires all model fields to have matching DB columns
- Chart.svelte uses tree-shaken ECharts — must register new components (e.g., LegendComponent) and add to ECOption type union
- ECharts markLine label.position requires literal type, not plain `string`
